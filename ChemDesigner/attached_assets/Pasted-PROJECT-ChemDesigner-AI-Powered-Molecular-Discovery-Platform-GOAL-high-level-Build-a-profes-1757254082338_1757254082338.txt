PROJECT: ChemDesigner — AI-Powered Molecular Discovery Platform

GOAL (high-level)
Build a professional, production-ready full-stack molecular discovery platform (MVP->Scale) using Flask backend + advanced AI models + rich frontend UI. This must be more than a demo: it should be a working prototype that implements analysis, generation (GraphVAE/MolGAN), RL-based optimization scaffolding, 3D / 2D visualization, persistence, and export/reporting.

TECH STACK (required)
- Backend: Python 3.11+, Flask, RDKit (rdkit-pypi), PyTorch (cpu), PyTorch Geometric OR fallback lightweight graph generator, Stable-Baselines3 (PPO) for RL scaffolding, SQLite for persistence.
- Frontend: Vanilla JS + modern bundler (Vite) or React (if faster to deliver), 3Dmol.js for 3D rendering, RDKit.js or 2D SVG fallback.
- Dev / Run: Single Repl using Nix (replit.nix) with a run command launching both backend and frontend dev servers. Provide Dockerfile and docker-compose for non-Replit deploy.

MUST-HAVE DELIVERABLES (priority order)
1. ✅ Core Flask APIs:
   - GET /api/health
   - POST /api/analyze  -> accepts SMILES, InChI, or plain chemical formulas (e.g., CO2, H2O) and returns validated SMILES + descriptors (MW, LogP, QED, SA, HBD, HBA, TPSA, rotatable bonds) + Lipinski summary + canonical formula.
   - POST /api/generate -> accepts latent vectors or generation parameters; returns SMILES + properties.
   - POST /api/optimize -> RL scaffold (PPO training stub + local policy apply) with configurable reward function; returns optimized SMILES candidates + metrics.
   - POST /api/save -> save molecule + metadata to SQLite; GET /api/list returns saved molecules.

2. ✅ Formula pre-processor:
   - Implement robust formula→SMILES mapper for common formulas and auto-translate Unicode subscripts (₂→2).
   - Canonicalize input with RDKit’s `MolToSmiles()` to ensure consistency.

3. ✅ Visualization:
   - Integrate 3Dmol.js for interactive 3D; fallback to RDKit.js or 2D SVG rendering.
   - Display molecule structure near properties panel.

4. ✅ Generation & ML:
   - Provide working GraphVAE model code (models/graph_vae.py) and scripts to run training on local dataset. If PyG installation fails on Replit, provide a functional lightweight generator stub that produces chemically-valid SMILES and mark it clearly as "stub — replace with PyG model".
   - Provide scripts/train.py to train VAE and train RL (PPO) using MolecularEnv; training may be configured to run locally/offloaded to another machine/GPU.

5. ✅ Persistence & Exports:
   - SQLite DB with tables: analyses, generated_molecules, runs.
   - Export results to CSV and a one-page PDF report generator (reportlab or wkhtmltopdf fallback).

6. ✅ Frontend:
   - Single-page app with pages: Home, Generate (latent explorer + sliders), Analyze (SMILES/formula input), Evaluate (batch stats + saved results).
   - Clean UI, accessible controls, copyable SMILES, download buttons.
   - Build script writes production build to backend/static.

7. ✅ Developer convenience:
   - Provide replit.nix (or clear instructions) and .replit run command.
   - Provide README with exact commands for development and production docker-compose.
